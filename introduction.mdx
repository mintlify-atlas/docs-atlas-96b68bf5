## What is Svelte ASCII Art?

Svelte ASCII Art is a Svelte 5 component that renders ASCII art as scalable SVG graphics. It preserves the monospace character grid while providing modern web capabilities like infinite scaling, custom styling, and export functionality.

<Note>
Requires Svelte 5.0.0 or higher. The package is built specifically for Svelte 5's new component architecture.
</Note>

## Key features

### SVG rendering

The component renders each character as an SVG `<tspan>` element positioned on a precise coordinate grid. The result is:

- Infinitely scalable without quality loss
- Sharp rendering at any resolution
- Support for custom fonts via CSS
- Efficient DOM structure using grouped text elements

### Grid mode

Enable grid overlays to visualize the character grid:

```svelte
<AsciiArt {text} grid gridClass="ascii-grid" />
```

Perfect for:
- Debugging alignment issues
- Creating technical diagrams
- Educational purposes
- Pixel art style graphics

### Frame and margin control

Define a frame area with configurable margins:

```svelte
<AsciiArt {text} frame margin={[1, 2]} frameClass="ascii-frame" />
```

The margin system uses grid cells as units, making it intuitive to work with character-based layouts.

### Export functionality

Export your ASCII art with embedded styles:

```typescript
import { exportSvg, exportSvgToPng } from 'svelte-asciiart';

// Export as standalone SVG
const svgMarkup = exportSvg(svgElement, {
  includeBackground: true,
  backgroundColor: '#f3f4f6'
});

// Export as PNG with 2x resolution
const pngDataUrl = await exportSvgToPng(svgElement, {
  includeBackground: true,
  backgroundColor: '#f3f4f6',
  scale: 2
});
```

The export functions extract computed CSS styles and embed them directly in the SVG, making the output portable and self-contained.

## Browser support

Svelte ASCII Art works in all modern browsers that support:
- SVG 1.1
- CSS custom properties (CSS variables)
- Modern JavaScript (ES2020+)

For PNG export functionality, the browser must support the Canvas API.
