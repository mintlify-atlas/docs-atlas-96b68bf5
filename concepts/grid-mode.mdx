---
title: 'Grid Mode'
description: 'Character-by-character alignment with visible grid lines'
---

Grid mode adds visible gridlines to your ASCII art, making the character-by-character structure explicit. This is useful for creating diagrams, aligning elements, or debugging layouts.

## Enabling Grid Mode

Enable grid mode with the `grid` prop:

```svelte
<AsciiArt 
  text="Hello\nWorld" 
  grid={true}
/>
```

<Tip>
  Combine grid mode with `frame={true}` to create bordered diagrams with visible cell alignment.
</Tip>

## How Grid Lines are Rendered

Grid lines are rendered as a single SVG `<path>` element with multiple drawing commands:

```svelte
{#if grid}
  <path
    class={gridClass}
    d={[
      ...totalGridCols.map((c) => `M ${fmt(c * cellAspect)} 0 V ${fmt(totalRows * cellHeight)}`),
      ...totalGridRows.map((r) => `M 0 ${fmt(r * cellHeight)} H ${fmt(totalCols * cellAspect)}`)
    ].join(' ')}
    fill="none"
  />
{/if}
```

From `AsciiArt.svelte:102-111`

### Path Construction

The grid is created using SVG path commands:

- **Vertical lines**: `M x 0 V height` - Move to (x, 0), draw vertical line to height
- **Horizontal lines**: `M 0 y H width` - Move to (0, y), draw horizontal line to width

<Info>
  All grid lines are combined into a single `<path>` element for optimal rendering performance.
</Info>

## Grid Dimensions

The grid spans the entire viewBox, including margins:

```typescript
const totalCols = frameCols + parsedMargin.left + parsedMargin.right;
const totalRows = frameRows + parsedMargin.top + parsedMargin.bottom;
const totalGridCols = Array.from({ length: totalCols + 1 }, (_, c) => c);
const totalGridRows = Array.from({ length: totalRows + 1 }, (_, r) => r);
```

From `AsciiArt.svelte:73-78`

<Note>
  Grid lines are drawn at cell boundaries, so a 5-column grid has 6 vertical lines (0, 1, 2, 3, 4, 5).
</Note>

### Example Calculation

For a 5×2 character grid with 1-unit margins:

```
Frame: 5 cols × 2 rows
Margins: 1 on all sides
Total: 7 cols × 4 rows (including margins)
Grid lines: 8 vertical × 5 horizontal
```

## Styling Grid Lines

Customize grid appearance using the `gridClass` prop:

```svelte
<AsciiArt 
  text="Box\nArt" 
  grid={true}
  gridClass="custom-grid"
/>

<style>
  :global(.custom-grid) {
    stroke: #3b82f6;
    stroke-width: 0.02;
    opacity: 0.5;
  }
</style>
```

<Accordion title="Why use :global()?">
  Svelte scopes styles by default. The `:global()` wrapper makes the style apply to elements outside the current component's scope, including the SVG elements rendered by AsciiArt.
</Accordion>

### Common Grid Styles

<CodeGroup>

```css Subtle Grid
.subtle-grid {
  stroke: currentColor;
  stroke-width: 0.01;
  opacity: 0.2;
}
```

```css Dotted Grid

  stroke-width: 0.02;
  stroke-dasharray: 0.02 0.1;
  opacity: 0.4;
}
```

```css Bold Grid

```

## Grid Cell Alignment

Characters are positioned within grid cells with small offsets for visual centering:

```typescript
// Horizontal offset: 10% of cell width
<tspan x={fmt(offsetX + c * cellAspect + 0.1 * cellAspect)}>

// Vertical offset: 80% of cell height (baseline alignment)
<text y={fmt(offsetY + r * cellHeight + 0.8 * cellHeight)}>
```

From `AsciiArt.svelte:126-132`

### Visual Representation

```
┌─────┬─────┬─────┐
│  H  │  e  │  l  │  ← Characters centered in cells
├─────┼─────┼─────┤
│  l  │  o  │     │
└─────┴─────┴─────┘
  ↑
  Grid lines at cell boundaries
```

<Tip>
  The 0.1 horizontal and 0.8 vertical offsets are tuned for most monospace fonts. You cannot customize these offsets directly, but you can adjust overall positioning using the `margin` prop.
</Tip>

## Grid with Cell Aspect

The `cellAspect` prop affects grid cell dimensions:

```svelte
<AsciiArt 
  text="A" 
  grid={true}
  cellAspect={0.6}
/>
<!-- Cell: 0.6 units wide × 1 unit tall (rectangular) -->

<AsciiArt 
  text="A" 
  grid={true}
  cellAspect={1.0}
/>
<!-- Cell: 1 unit wide × 1 unit tall (square) -->
```

## Combining Grid with Frame

Use both `grid` and `frame` to create bordered diagrams:

```svelte
<AsciiArt 
  text="Diagram\nContent" 
  grid={true}
  frame={true}
  margin={1}
  gridClass="diagram-grid"
  frameClass="diagram-frame"
/>

<style>
  :global(.diagram-grid) {
    stroke: #e5e7eb;
    stroke-width: 0.01;
  }
  
  :global(.diagram-frame) {
    stroke: #1f2937;
    stroke-width: 0.05;
  }
</style>
```

<Info>
  The grid extends to margins, while the frame only surrounds the content area. This creates a clear visual hierarchy.
</Info>

## Performance Considerations

<Info>
  Grid rendering is highly efficient because all lines are combined into a single `<path>` element.
</Info>

Performance characteristics:

- **Single DOM element**: All grid lines in one `<path>`
- **No JavaScript calculation**: Grid path is derived reactively
- **Minimal impact**: ~1-2ms additional render time for typical grids
- **Scales well**: 100×100 grid renders as fast as 10×10

## Grid Path Structure

For a 3×2 grid, the generated path looks like:

```
M 0 0 V 2 M 0.6 0 V 2 M 1.2 0 V 2 M 1.8 0 V 2 M 0 0 H 1.8 M 0 1 H 1.8 M 0 2 H 1.8
```

Broken down:
- `M 0 0 V 2` - Vertical line at x=0 from y=0 to y=2
- `M 0.6 0 V 2` - Vertical line at x=0.6 from y=0 to y=2
- `M 0 0 H 1.8` - Horizontal line at y=0 from x=0 to x=1.8
- And so on...

<Tip>
  Understanding the path structure can help you create custom grid styles using CSS `stroke-dasharray` for dotted or dashed lines.
</Tip>
