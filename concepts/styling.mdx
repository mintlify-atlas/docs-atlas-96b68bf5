The `AsciiArt` component provides flexible styling options through CSS classes, CSS variables, and standard SVG attributes.

## CSS Variables

The component uses CSS variables for easy theming without prop drilling.

### Font Family

The font family is controlled via the `--ascii-font-family` CSS variable:

```typescript
style="width: 100%; height: 100%; font-family: var(--ascii-font-family, {defaultFontStack});"
```

From `AsciiArt.svelte:100`

Default fallback:
```typescript
const defaultFontStack = 'ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace';
```

From `AsciiArt.svelte:69-70`

<Tip>
  Set `--ascii-font-family` on a parent element to change the font for all AsciiArt components within.
</Tip>

### Custom Font Example

## Custom Classes

The component accepts class props for grid and frame elements.

### Grid Class

Style grid lines using the `gridClass` prop:

```svelte
<AsciiArt 
  text="Grid" 
  grid={true}
  gridClass="custom-grid"
/>

<style>
  :global(.custom-grid) {
    stroke: #3b82f6;
    stroke-width: 0.02;
    stroke-dasharray: 0.05 0.05;
    opacity: 0.5;
  }
</style>
```

<Note>
  Use `:global()` in Svelte to style elements rendered by the AsciiArt component, as they're in a different component scope.
</Note>

### Frame Class

Style the frame rectangle using the `frameClass` prop:

```svelte
<AsciiArt 
  text="Framed" 
  frame={true}
  frameClass="custom-frame"
/>

<style>
  :global(.custom-frame) {
    stroke: #ef4444;
    stroke-width: 0.1;
    stroke-linecap: round;
    stroke-linejoin: round;
  }
</style>
```

## SVG Attributes

The component spreads additional props to the root `<svg>` element:

```typescript
interface Props extends SVGAttributes<SVGSVGElement> {
  text: string;
  rows?: number;
  cols?: number;
  // ...
}

let { text, rows, cols, /* ... */, ...rest }: Props = $props();
```

From `AsciiArt.svelte:7-36`

This allows you to pass any valid SVG attribute:

```svelte
<AsciiArt 
  text="Styled" 
  class="my-svg"
  id="ascii-art-1"
  style="background: #f3f4f6;"
  aria-label="ASCII art rendering"
/>
```

<Info>
  SVG attributes like `class`, `style`, `id`, and ARIA attributes are passed directly to the `<svg>` element.
</Info>

## Style Extraction for Export

When exporting to SVG or PNG, styles are automatically embedded using the utility functions:

```typescript
function extractClassStyles(svgEl: SVGSVGElement): string {
  const styleRules: string[] = [];
  const processedClasses = new Set<string>();
  const elementsWithClasses = svgEl.querySelectorAll('[class]');
  
  elementsWithClasses.forEach((el) => {
    const classAttr = el.getAttribute('class')!;
    const classes = classAttr.split(/\s+/).filter(Boolean);
    
    classes.forEach((className) => {
      if (processedClasses.has(className)) return;
      processedClasses.add(className);
      
      const computed = getComputedStyle(el);
      const declarations: string[] = [];
      
      SVG_STYLE_PROPS.forEach((prop) => {
        const value = computed.getPropertyValue(prop);
        if (!value || (value === 'none' && prop !== 'fill')) return;
        declarations.push(`${prop}: ${value}`);
      });
      
      if (declarations.length) {
        styleRules.push(`.${className} { ${declarations.join('; ')} }`);
      }
    });
  });
  
  return styleRules.join('\n');
}
```

From `utils.ts:21-54`

<Info>
  Styles are extracted from mounted DOM elements, so the SVG must be rendered in the browser before exporting.
</Info>

## Responsive Sizing

The SVG is styled to be responsive by default:

```typescript
style="width: 100%; height: 100%; /* ... */"
```

<Accordion title="How does responsive sizing work?">
  The SVG has intrinsic dimensions set via `width` and `height` attributes, but the inline style overrides these with `width: 100%; height: 100%`. This makes the SVG fill its container while the `preserveAspectRatio="xMinYMin meet"` attribute ensures the aspect ratio is maintained.
</Accordion>

### Container-Based Sizing

```svelte
<div style="width: 400px; height: 300px;">
  <AsciiArt text="Fits container" />
</div>
```

The AsciiArt component will fill the 400×300 container while maintaining its aspect ratio.

### Fixed Size

```svelte
<AsciiArt 
  text="Fixed" 
  style="width: 200px; height: 100px;"
/>
```

<Tip>
  Override the default `width: 100%; height: 100%;` style to set fixed dimensions.
</Tip>

## Advanced Styling Techniques

### Text Shadow

```svelte
<AsciiArt 
  text="Shadow" 
  style="filter: drop-shadow(2px 2px 4px rgba(0,0,0,0.3));"
/>
```

### Gradient Text

```svelte
<svg>
  <defs>
    <linearGradient id="textGradient">
      <stop offset="0%" stop-color="#3b82f6" />
      <stop offset="100%" stop-color="#8b5cf6" />
    </linearGradient>
  </defs>
</svg>

<AsciiArt 
  text="Gradient" 
  style="fill: url(#textGradient);"
/>
```

<Note>
  When using `fill` in inline styles, it overrides the default `fill="currentColor"` from the text elements.
</Note>

### Animations

```svelte
<AsciiArt 
  text="Animated" 
  class="animated-ascii"
/>

<style>
  :global(.animated-ascii) {
    animation: pulse 2s ease-in-out infinite;
  }
  
  @keyframes pulse {
    0%, 100% { opacity: 1; }
    50% { opacity: 0.5; }
  }
</style>
```

### Stroke Text Effect

```svelte
<AsciiArt 
  text="Outlined" 
  style="
    fill: white;
    stroke: black;
    stroke-width: 0.02;
    paint-order: stroke fill;
  "
/>
```

<Tip>
  The `paint-order: stroke fill` property ensures the stroke is rendered behind the fill, creating a clean outline effect.
</Tip>
